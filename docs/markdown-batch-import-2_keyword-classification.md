## Plan: Markdown Batch Import (2) - Keyword Classification

TL;DR: 分割された各セクションからキーワードを検出し、自動的に `category`（食事、環境、身体など）と `status`（4象限）の候補を割り当てます。これにより、ユーザーが各付箋を一枚ずつ分類する手間を最小限にします。

**Steps**
1. **キーワード辞書の作成**
   - カテゴリキーワード（例：「食事」「嚥下」→ `food`、「手すり」「段差」→ `environment`）。
   - 状態キーワード（例：「できる」「自立」→ `can`、「転倒」「危ない」→ `risk`）。
2. **検知ロジックの実装**
   - 正規表現や単純なマッチングを用いて、付箋のタイトルと内容をスキャン。
   - スコアリングを行い、最も確度の高いカテゴリを初期値として設定。
3. **提案 UI の実装**
   - 分類された付箋を「確定」する前に、ユーザーが確認・修正できるプレビュー画面を表示。
   - 自動分類が確実でない場合は「未分類」としてマーク。

**Relevant files**
- `src/utils/keywordClassifier.ts` (新規: 判定ロジック)
- `src/components/pending/PendingNoteItem.tsx` (自動分類の結果表示)
- `src/types/index.ts` (自動分類の状態フラグ追加)

**Verification**
1. 「食事について」という見出しのセクションが自動的に `food` カテゴリに割り当てられること。
2. 「ふらつき」を含む内容が `risk` ステータスに提案されること。
3. ユーザーが提案された分類を手動で上書きできること。
